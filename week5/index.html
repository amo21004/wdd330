Notes

  • Exceptions are errors that return a value and can be captured by the programmer and dealt accordingly.
  • Exceptions produce a stack trace which can be used to debug the problem. The stack trace shows a list of functions calls that resulted in the error.
  • ES5 introduced "Strict Mode" which enforces certain good practices. One such enforcement is that if an undeclared variable is assigned or referenced, it will produce a reference error.
  • Modules by default use "Strict Mode" and there is no way to disable this.
  • The alert function and console.log method are very useful while debugging.
