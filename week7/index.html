Ch11: Further Functions

  • Functions that are expensive to run can make use of Memoization which stores the result of a call in a cache. Future calls will return the data from the cache.
  • IIFE - These are anonymous functions which are invoked immediately after the function expression.
  • IIFEs can be useful when you want to restrict the scope of variables. A variable that is declared inside an IIFE is declared when its invoked and is destroyed immediately after the execution of that function ends.
  • Functions in JavaScript can both define and redefine themselves.
  • JavaScript supports async functions that execute asynchronously.
  • JavaScript supports Closures however Closures in JavaScript are not the same as Closures found in PHP.

Ch13: AJAX

  • AJAX stands for Asynchronous JavaScript and XML. In the past, AJAX calls usually returned a response in XML format however now its more common for them to return a response using JSON.
  • When using AJAX, it is important to understand Same-Origin Policy and CORS.
  • AJAX requests are now usually handled using the Fetch API. The fetch method returns a promise.
